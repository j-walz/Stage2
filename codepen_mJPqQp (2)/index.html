Notes From Stage0

How the Internet Works

The internet is the worlds largest network of computers. Basically it is just a web for computers
to talk to other computers, but on a giant scale. The user will get on a browser and send a request
over the internet to a web server that is hosting what ever information the user wants. The web server
then repsonds to the user with the information request and sends it back across the internet to appear on the user's browser.

HTML

HTML is the basic language of the internet. Almost all of the web pages on the internet are written using HTML.
HTML documents can contain links, images, text, video, and much more. The way that these different elements
of an HTML document are written into the document are with tags. Tags are markers that notify the HTML document
that a specified set of text is not just text but an insrtuction of how to treat other sets of text. Tags are surrounded by < and >
with the closing tag having a / infront of it. This is so the HTML document knows how to treat that text, and then the tags
themselves surround the bit of text that they are affecting. An example of a tag is the b tag, which tells the HTML document
that all of the text inbetween the opening and closing b tags should be bold.

Together a tag and the text that thattag is affecting are called and element. There are two types of elements, inline
and block. Inline elements keep the element within the line of text and do not add anything extra to the text.
Block elements put an invisible box around the text that the writer can use to set that bit of text apart from others.

Thinking Like a Computer

Computers do exactly what they are told, but what they are told is not always what is meant. To have think like
a computer is to be able to think in systematic steps. One must take everything one step at a time and without
assuming any intetnion that may seemt to be built into the intrusction. This can be fairly difficult and requires much practice.

The Document Object Model

The Document Object Model, or DOM, is how web browsers interperit html files like this one. An html page is not written in some bizaar language, rather it is just written using a particular structure that allows browsers to understand what the author intends. The DOM is a branched structure, having elements branch to other elements which may in turn branch off further and further. Not only does this allow for browers to read html documents it is also fairly easliy understood by the human eye.

Developer Tools

The broswer Google Chrome has a usefull feature in the Developer Tools. They allow the user to inspect the elements of what ever web page he or she is viewing. If a person who is starting out in fron end web development is stuck or sees an interesting feature on a web page, than that person can utilize the Developer Tools to see what kind of tag, css, and or java script was used to produce the desired effect.

Text Editors

Text Editors are a great tool for programmers. They help the programmer by keeping him or her in line with the DOM and grammatical structure of what ever language is being used

HTML and Boxes

When one is looking at a web page, especially a good one, many shapes and design features will become apparent. However, looks may be deceiving. All of the elements display on an HTML page are actually boxes. Though one may see a circle or a triangle, those shapes are contained in boxes. This makes it much easier for prgrammers to plan and lay out a web page as well as rearrange it if neccesary.

Avoid Repition

Writing a program or creating a web page can be very time consuming and tedious. Becuase of that fact it is important for programmers to avoid repition as much as possible so as not to add to their already burdensome task. In avoiding rewriting the same code over and over consistency and the ability to change large portions of code are gained. By reusing a trusted piece of code programmers can be assuered that what they are writing will be good.

CSS

CSS, or Cascading Style Sheets, is the language used to add style to HTML web pages. With CSS a programmer can take a bland and uniteresting web page and turn it into a colorful and attention grabbing web page.

Notes from Stage 2

The Computer as a Machine

Most machines are built with only a few specific functions and abilities in mind. With out physically altering the machine it can only prefomr the tasks that it was built to do. In this regard the computer is a special machine. It was built such that it can be repurposed over and over to preform almost any task, including controling other machines. It has this ability through computer programs.

Computer Programs

A computer program is a detailed list of instructions write so that a computer can understand them. The computer program must follow a precise sequence of steps while written in a programming language which is run through an interpreture so that the computer can understand it.

Programming Language

Programming languages are languages that have a strict grammar and syntax which eliminate all ambiguity. It is very important that a programming language is very precise in it's message to the computer. The computer can understand implied meaning, nor can it understand what a person might have meant if he or she misspeaks. A program must inform the computer exaclty what the computer needs to do, this is why it is important for the programming language to have a strict and precise grammar.

Python

Python is an example of a programming language. When run through a python interpreture it can tell the computer what to do. It is a fairly powerful and popular language. The program is consisted of expressions which are statements in the Python language. An example of a Python expression is 'print 1 + 1'. That expression is valid because it follows Python's grammar and syntax.

Variables

In programming variables are what stores values or expressions. A variable can be any string of alpha characters or numbers so long as it does not start with a number. An example of a variable being set is name = "jacob" where name is a variable storing the value jacob. When assinging variables, or giving variables the value they are to store, the '=' symbol does not mean 'equal to' as it does in arithmatic. Rather the '=' symbol should be seen more as an arrow pointing from left to right that transfers the value into the variable.

Variables can be very useful as a way to capture as yet unknown values as well as giving names to values so that they may be better understood by someone reading the code. Variables can be given values of intergers, which are numbers, or strings, which are alpha characters. For something to be a string it must be in either a set of single quotes or a set of double quotes. This extends to numbers as well, meaning that "2" is a string and not a number.

Functions

A function is a routine that takes input, either from the user or from other parts of the program, and produces outputs. To write a function one start with the keyword 'def' which can be though of define. One then writes the functions name followed by a set of (). Inside the () are any number of parameters or inputs, including zero, that the function will need. Finally a colon at the end of the line. The next line and all subsequent lines of the function should be indented as to show that they are a part of that function. The lines that follow are the meat of the function, they are defining what the function will do with the input and what output it will produce.

Just defining a function will not do anything, for the function to be effective the program must call it. To call a function it just need be written out followed by the values desired to fulfill all of the requried parameters. Functions are a great tool for reducing repitition since when one is written it can be used over and over just as once one makes a hammer one does not need to make another hammer if the need arises.

Loops

Loops are a very useful tool for the programmer. They allow for a specific section of code to be perfomred a specifc amount of times. The two main types of loops are the 'while' loops and the 'for' loops. The while loop is started with True condition and continues until the condition is Fales. If that condition never becomes Fales the 'while' loop, and the program, continues running forever, which makes while loops potentially very dangerous. 'For' loops are structured with a finite number of iterations so that they cannot run forever.

Solving Problems

At it's simplest writing code is a series of solving problems. With that being the case, it is of great importance that a programmer have a strong strategy for solving problems. The first thing to do when presented a problem is make sure that you understand what the problem is asking, and what it expects. You need to know the inputs and expected outputs. Once you have a grasp of the what the problem is about then you can start trying to solve it.

A good idea when starting to solve the problem is to break the problem down into smaller problems and write out pseudo code to solve those smaller problems. Once you have some of the smaller problems solved with psuedo code, write the actuall code and test it. Keep repeating that process until the whole problem is solved.
